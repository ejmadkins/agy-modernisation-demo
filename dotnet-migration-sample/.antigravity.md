# Antigravity Orchestration - .NET Modernization

This file orchestrates the step-by-step modernization of the Contoso University application from legacy .NET 5 / Entity Framework 6 with SQL Server, to a containerized .NET 8 / EF Core application with PostgreSQL.

## Workflow

1.  **Planning and Research**:
    *   Examine the legacy codebase, noting references to `EntityFramework` (EF6) in `ContosoUniversity.csproj`, references to SQL Server connections, and startup/middleware setup.
    *   Synthesize a comprehensive design plan in `implementation_plan.md` (or directly execute if approved).
    *   Leverage the `.agents/skills/dotnet-modernizer` skill for detailed guidance on code transformations.

2.  **Code Refactoring (Application Layer)**:
    *   **Project File Modernization**: Convert `ContosoUniversity.csproj` to the modern SDK-style format targeting `net8.0`. Remove old EntityFramework references and add `Npgsql.EntityFrameworkCore.PostgreSQL`, `Microsoft.EntityFrameworkCore.Tools`, and `Microsoft.AspNetCore.Mvc.NewtonsoftJson`.
    *   **Database Refactoring**:
        *   Migrate `SchoolContext` from `System.Data.Entity.DbContext` to `Microsoft.EntityFrameworkCore.DbContext`.
        *   Convert old model mappings and conventions to EF Core (e.g. replacing pluralizing conventions with explicit table names `ToTable()`).
        *   Implement PostgreSQL concurrency handling using the PostgreSQL `xmin` system column.
        *   Replace old SQL Server database initializers/seeders with a dedicated, modernized `DbInitializer` and seed configuration.
    *   **Dependency Injection Configuration**:
        *   Merge configurations from `Startup.cs` into the modern `Program.cs` file.
        *   Register `SchoolContext` with `AddDbContextPool` using the Npgsql provider.

3.  **Google Cloud Run Container Contract Compliance**:
    *   **Port Binding**: Configure Kestrel to bind to `0.0.0.0` on the port specified by the `PORT` environment variable (defaulting to 8080 if undefined).
    *   **Structured Logging**: Configure `ILogger` to output structured console JSON logs which Google Cloud Logging can ingest and parse automatically.
    *   **Graceful Shutdown**: Hook into `IHostApplicationLifetime.ApplicationStopping` to handle `SIGTERM` signals and close active database connections and requests cleanly.

4.  **Containerization**:
    *   Create a optimized multi-stage `Dockerfile` (using `.net 8.0 SDK` to compile and `aspnet:8.0` for runtime).
    *   Write a deprecation-free `compose.yaml` running the application container and a healthchecked PostgreSQL container.

5.  **Documentation**:
    *   Generate a detailed `MODERNIZATION.md` report detailing the modernized application structure, endpoints table, local validation procedures, and verification results.
